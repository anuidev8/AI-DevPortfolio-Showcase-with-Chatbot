- `portfolio.md`   → Portfolio content rules: projects, services, skills, GitHub snippets, tech section
- `ux.md`          → UX rules & patterns for this portfolio (interactions, responsiveness, accessibility)
- `ui-elements.md` → Design system: colours, typography, components, animations, layout patterns

## Usage

- For chatbot/assistant features → `chatbot.md`.
- For portfolio content & sections → `portfolio.md`.
- For behaviour & layout decisions → `ux.md`.
- For visual details, animations, component specs → `ui-elements.md`.
- For implementation (libraries, patterns, file placement) → `stack.md`.
- For external links & references → `references.md`.

## Rules

- If a rule is missing from a knowledge file:
  - Ask the human to update the appropriate topic file.
  - Do not invent new rules or design decisions.
- Knowledge files are the single source of truth for this project's domain.
- TASKS.md is the single source of truth for what work is planned or in progress.
